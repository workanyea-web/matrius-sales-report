# Описание проекта: Matrius Sales Report Bot

**Реальная рабочая задача:** ежедневный отчёт по воронке продаж Matrius в Telegram.

Сейчас отчёт руководитель составляет вручную: смотрит на лиды в amoCRM, считает конверсию, пишет итог в чат. Задача — автоматизировать это через мультиагентную систему в Claude Code.

---

## Проектирование по структуре урока 10

### 1. Что мы хотим получить на выходе?
Готовое Telegram-сообщение с ежедневным отчётом:
- сколько лидов на каждом этапе воронки
- конверсия по переходам
- топ-менеджер дня
- одна ключевая рекомендация на завтра

### 2. Какие этапы нужны?
1. Принять входные данные (лиды по статусам + менеджеры)
2. Посчитать конверсию по каждому переходу воронки
3. Выявить аномалии и лучшего менеджера
4. Сформировать структурированный отчёт
5. Отформатировать текст для Telegram

### 3. Какие навыки и инструменты нужны для каждого этапа?
| Этап | Инструменты |
|------|------------|
| Приём данных | Read (JSON-файл или ввод пользователя) |
| Расчёт конверсии | Аналитика (Claude) |
| Поиск аномалий | Аналитика (Claude) |
| Формат JSON | Write |
| Telegram-текст | Только чтение + генерация |

### 4. Какие этапы отдать субагентам?
- **Главный агент (скилл `sales-report`)** — принимает данные, запускает субагентов, возвращает итог
- **@data-analyst** — этапы 2–3: считает конверсию, находит топ-менеджера, выявляет аномалии, возвращает JSON
- **@report-writer** — этап 4–5: принимает JSON, форматирует Telegram-сообщение с эмодзи и структурой

### 5. Как один агент передаёт результат следующему?
Контракт между агентами — JSON:
```json
{
  "date": "2026-06-01",
  "funnel": {
    "new_leads": 12,
    "contacted": 9,
    "trial_scheduled": 6,
    "trial_done": 5,
    "paid": 2
  },
  "conversions": {
    "contact_rate": "75%",
    "trial_rate": "67%",
    "close_rate": "40%"
  },
  "top_manager": "Мария",
  "anomalies": ["Конверсия в оплату ниже нормы (норма 50%)"],
  "recommendation": "Проработать этап после пробного урока — 3 клиента не дошли до оплаты"
}
```

### 6. Как проверяется качество финального результата?
Отчёт считается корректным, если содержит:
- ✅ дату
- ✅ данные по всем 5 этапам воронки
- ✅ имя топ-менеджера
- ✅ хотя бы одну аномалию или положительный вывод
- ✅ конкретную рекомендацию (не общую фразу)

## Архитектура системы

```
Пользователь вводит данные
        ↓
sales-report (скилл-оркестратор)
        ↓
   @data-analyst (субагент)
   — считает конверсию
   — находит топ-менеджера
   — выявляет аномалии
   — возвращает JSON
        ↓
   @report-writer (субагент)
   — читает JSON
   — форматирует Telegram-сообщение
        ↓
Готовый текст отчёта
```

**Модель взаимодействия:** последовательная + ролевая (два субагента с чёткими ролями)
